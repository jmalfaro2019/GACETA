KAFKA SETUP INSTRUCTIONS - Windows
===================================

Prerequisites: Download and extract Apache Kafka to your system.
Change to the Kafka installation directory before running these commands.

STEP 1: Format Kafka Cluster (First time setup only)
=====================================================
NOTE: Each system will automatically generate its own Cluster UUID.
Simply run the command below (no need to copy the UUID from another machine):

.\bin\windows\kafka-storage.bat format -c .\config\kraft\server.properties

Alternative (if you want to specify a custom UUID):
.\bin\windows\kafka-storage.bat format -t <YOUR_CLUSTER_UUID> -c .\config\kraft\server.properties
(Replace <YOUR_CLUSTER_UUID> with a unique identifier if needed)


STEP 2: Start Kafka Server
==========================
Run this in a dedicated terminal window:

.\bin\windows\kafka-server-start.bat .\config\kraft\server.properties


STEP 3: Create Topic "gacetas_nuevas"
======================================
Run this in another terminal (one time setup):

.\bin\windows\kafka-topics.bat --create --topic gacetas_nuevas --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1


STEP 4a: Start Producer (for testing/sending messages)
======================================================
.\bin\windows\kafka-console-producer.bat --topic gacetas_nuevas --bootstrap-server localhost:9092


STEP 4b: Start Consumer (for monitoring incoming messages - OPTIONAL)
====================================================================
Run this in another terminal to verify the producer is sending messages:

.\bin\windows\kafka-console-consumer.bat --topic gacetas_nuevas --from-beginning --bootstrap-server localhost:9092


NOTES:
- Keep the Kafka server running at all times
- The producer and consumer are for testing purposes
- The main worker_procesador.py will handle consuming messages from the topic
- Use --from-beginning flag to read all existing messages from the topic start
- Each machine will auto-generate its own Cluster UUID - no need to match other systems
